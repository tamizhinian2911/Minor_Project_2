# Minor_Project_2
##SpendDNA

SpendDNA is a financial intelligence parser and profile engine that transforms raw, erratic bank/UPI transaction exports into a data-driven year-end story. Styled as a "Spotify Wrapped for your wallet," it standardizes cross-platform transaction logs, segments spending habits by category, isolates behavioral anomalies, and profiles the user via algorithmic behavioral archetypes.

🛠️ The "Hard Mode" ConstraintsTo showcase algorithmic efficiency and deep proficiency in raw data handling, the codebase is engineered under strict execution limits:Allowed: Core Python, foundational Pandas (vectorized string and datetime operations, basic groupby/pivot_table), and NumPy.  Banned: pandas-profiling, sweetviz, scikit-learn, regex (re), collections.Counter, and all visualization libraries (matplotlib, seaborn, plotly). Every metric, text distribution bar, and mathematical classification engine was constructed from scratch.  📊 Core Features & Engineering Architecture1. Multi-Format Transaction ParserBank and UPI statements rarely adhere to a single standard. The parser handles data normalization end-to-end:Heterogeneous Date Parsing: Vectorizes and sequences parsing across 4 distinct datetime formats (%d/%m/%y, %Y-%m-%d, %d-%b-%y, %d %b %Y) without structural ambiguity.  Text Sanitation: Massages inconsistent numerical strings, cleansing local currency symbols (₹, Rs., commas) to safely cast transactions into numeric data types.  Deduplication: Runs rigorous compound field filtering to drop exact transaction clones while maintaining data integrity.  2. Hierarchical Keyword Vendor ExtractorWithout relying on regex, a sequential substring search array maps complex transaction descriptions (e.g., POS SWIGGY BANGALORE, BUNDL Tech P L) to canonical corporate entities. The priority queue places highly specific criteria (e.g., Amazon Prime) ahead of generic parents (Amazon) to prevent structural misclassification.  3. Contextual Anomaly Detection EngineInstead of standardizing anomalies across a global pool—which would incorrectly flag small purchases—the engine implements a Z-Score grouping transform:  $$Z = \frac{x - \mu_{\text{category}}}{\sigma_{\text{category}}}$$By calculating the category-specific mean ($\mu$) and standard deviation ($\sigma$), the framework dynamically isolates intra-category spikes (such as an unprecedented ₹2,000 food order) while ignoring structurally normal, high-ticket transactions like monthly investment capital.  4. Algorithmic Spending ArchetypesThe engine evaluates mathematical criteria built from user habits to assign specific behavioral identities:  THE SHOPAHOLIC: E-commerce spend outpaces total consumption by $>15\%$ (MATCH: 37.4% of spending).  THE INVESTOR: Investment outlays exceed $>15\%$ of total debits (MATCH: 16.4% of spending).  THE YOLO SPENDER: Savings rate collapses below a threshold of $10\%$ (MATCH: -229.3% savings rate).  THE BREWERY REGULAR: $>40\%$ of total restaurant footprint falls on Fri/Sat evenings (19:00–23:00) (No Match: evaluated at 14.7%).  📈 Key Insights & Production Highlights (From Rahul's Statement)The Consumption Picture: Out of 1,310 parsed transactions, E-commerce dominated at 37.4% (Rs. 564,577), followed closely by systematic Investments at 16.4% (Rs. 248,160).  Temporal Patterns: Behavioral habits clearly reflect lifestyle design—35.8% of Cafe transactions trigger precisely during morning commute hours (08:00–11:00), while 20.6% of Food Delivery runs occur during late-night hours (21:00–02:00).  Terminal Dashboard Interface: Features an automated text-based analytical dashboard rendering dynamic ASCII distribution charts using native string multiplication, matching visual value without a heavy visualization engine overhead.
